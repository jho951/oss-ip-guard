# Contributing

이 저장소에 기여하려면 아래 기준을 따르면 됩니다.

## 시작 전

- 기존 이슈나 문서를 먼저 확인합니다.
- `main`에는 직접 push하지 않고 작업 브랜치에서 PR로 반영합니다.
- 구현 변경은 관련 테스트와 문서를 함께 수정합니다.
- 보안 관련 변경은 `SECURITY.md` 절차를 따릅니다.

## 브랜치와 PR

- 작업은 항상 `main`에서 새 브랜치를 만들어 진행합니다.
- 브랜치 이름은 `feat/...`, `fix/...`, `chore/...`, `docs/...`처럼 변경 의도가 드러나게 작성합니다.
- 검증이 끝나면 `main` 대상으로 PR을 엽니다.
- release, publish, 저장소 설정 변경도 같은 방식으로 PR에서 먼저 검토합니다.

## 변경 원칙

- 작은 단위로 변경하고 테스트를 함께 업데이트합니다.
- 문서와 코드가 어긋나면 둘 다 같이 수정합니다.
- 모듈 설명과 구현 책임은 `README.md`와 `docs/modules.md` 기준을 따릅니다.

## 검증

- 로컬에서 `./gradlew test`를 실행합니다.
- CI가 `main` 대상 PR에서 통과하는지 확인합니다.
- CI 관련 변경은 `docs/test-and-ci.md`도 함께 확인합니다.

## 코드 스타일

- 기존 패키지 구조를 유지합니다.
- 외부에 노출되는 타입은 단순하고 예측 가능하게 유지합니다.
- 운영 환경에서 민감 정보가 로그에 남지 않도록 주의합니다.
